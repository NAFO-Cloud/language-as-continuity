# Language as Continuity — GitHub Pages package

This package contains a simple static site version of the article **Language as Continuity: A Practical Method for Transferring AI Roles Across Platforms**.

## Files

- `index.html` — primary public page
- `article.md` — markdown mirror of the article
- `robots.txt` — basic crawler instructions
- `sitemap.xml` — placeholder sitemap; update the URL before publishing
- `.nojekyll` — avoids Jekyll processing on GitHub Pages

## Suggested repository name

`language-as-continuity`

## Suggested GitHub Pages URL

- Project Pages: `https://YOUR-USERNAME.github.io/language-as-continuity/`
- User Pages alternative: `https://YOUR-USERNAME.github.io/` if you want this to be the whole site

## Publish steps

1. Create a new public GitHub repository.
2. Upload all files from this folder to the repository root.
3. In GitHub, open **Settings → Pages**.
4. Set the publishing source to your main branch and root folder.
5. Wait for GitHub Pages to publish the site.
6. Edit `index.html`, `robots.txt`, and `sitemap.xml` to replace `YOUR-USERNAME` with your actual GitHub username.
7. Optionally add the PDF file to the same repository and link it from the page.
